设计规范

版式选择规则

  • 有 has_strong_image=true 且 screenshot_worthy=true → 使用「默认」版式（满屏图 + hook 色背景）
  • 没有强图 → 使用「无图兜底」版式（C 风格 · hook 色条 + 衬线引言）
  • 今日还没编辑选出 pick → 使用「空状态」版式（昨日 pick 降饱和 + 顶部 pill）
  • 长按 / 悬停 → 「长按预览」叠层，露出价格、技术原理、状态等补充信息

Hook 颜色系统

weird_form 形态奇特
new_behavior 新行为
unexpected_combo 意外组合
quiet_real_problem 沉默解决
new_interaction 新交互
niche_depth 垂直深耕

字体规范

  • 主字体：系统 sans（中文优先 PingFang / 微软雅黑）
  • 引言 / 杂志感：var(--font-serif)（Georgia / 宋体）
  • 字重只用 400（正文） / 500（标题），不用 600/700
  • 大小写：句首大写，不用 ALL CAPS、不用 Title Case

固定元素

  • 顶栏：左 Today · M.DD，右 No.XXX（自动累加期号）
  • 底栏：左 WeeklyAI（或域名），右"看详情 →"
  • 所有 hero 卡共用同一份顶 / 底栏结构，仅内容部分按版式切换
