# WeeklyAI v2 PRD —「每日一个值得点进去的 AI 产品」

> 版本：v2（subtraction 后）
> 编辑：ChenJunHsu
> 日期：2026-05-30

---

## 一、问题陈述

当前的 WeeklyAI 把目标设成了"找到所有融资过的 AI 创业公司并按分数排序"，结果首页变成了一个 TechCrunch 镜像——展示的全是任何关注 AI 的人都早已看过的融资新闻。对于真正想从 AI 产品中获得灵感的人（小米创新团队、AI 从业者、AI 爱好者），这个产品没有提供任何他们在 ChatGPT/Twitter/Perplexity 上得不到的东西。

**问题不是产品多不多，而是值不值得点。**

---

## 二、目标用户

- **核心**：对 AI 感兴趣、每天会主动找 AI 内容看的人（小米团队 + 同类创新岗位 PM + AI 重度用户）
- **次要**：希望快速获取 AI 行业动态的轻度关注者

**统一画像**：会在 Twitter / 即刻 / 小红书刷 AI 内容的人，对"oh shit 这个有意思"有低阈值的好奇心。

---

## 三、目标 (Goals)

1. **每日有一个值得点进去的产品** — 用户每次打开都能看到一个会让他想截图分享的 AI 产品
2. **覆盖大厂 + 小作坊** — OpenAI 新功能、独立开发者的奇怪硬件都能上榜，只要它够"想点"
3. **编辑器（我）的品味就是护城河** — 系统积累的是"我和团队认为有意思"的画像，不是通用的"流行"
4. **新闻是补充消费，不是主菜** — 用户来主要是看产品发现，看完再顺便扫一眼新闻

---

## 四、非目标 (Non-Goals)

1. **不做 AI 产品全量数据库** — 不追求收录完整性，宁缺毋滥
2. **不做评分排行榜** — 删除 1-5 分体系，二元化为"值不值得点"
3. **不做融资追踪器** — 融资是上下文，不是入选标准
4. **不做算法推荐** — 编辑制驱动，不上行为模型
5. **不做 Swipe Card / 沉浸式探索** — 移动端 Tinder 式刷卡只是装饰，删掉
6. **暂不做用户反馈循环（👎 按钮）** — v3 再说

---

## 五、核心用户故事

- **作为 AI 产品 PM**，我想每天打开看到一个"oh shit"产品，这样我能在通勤路上获得新的灵感来源。
- **作为 AI 爱好者**，我想看到大厂没大肆宣传但很有意思的新能力，或独立开发者的奇怪 AI 硬件，这样我能发现 Twitter 主流之外的东西。
- **作为我自己（编辑）**，我想每天用一个简单的 CLI 工具，从 20-50 个候选里花 5 分钟挑出一个 hero 产品，这样我的品味能被系统每天用上而不是被埋在配额规则里。
- **作为读者**，我想在看完 hero 产品后顺手扫一眼 AI 新闻列表，这样我一次访问能完成"发现 + 跟进"两件事。

## 六、功能需求

### P0（必须有）

| 功能 | 验收标准 |
|---|---|
| **Hero 卡片** | 首页顶部展示一个超大产品卡：logo / 名称 / 一行 `why_matters` / `hook` 徽章 / 跳转详情 |
| **Recent picks 滚动条** | Hero 下方横向滚动最近 7 个产品（小卡片） |
| **AI 新闻流** | 现有 blog 模块直接复用，不动 |
| **二元产品标签** | 产品模型加 `screenshot_worthy: bool` + `hook: enum`，弃用 `dark_horse_index` |
| **抓取池重构** | Search prompt 改写——主池抓 Kickstarter / Indiegogo / Makuake / Show HN / Product Hunt / 小米有品 / 摩点；副池抓 OpenAI / Anthropic / Google / Xiaomi / 字节官方博客；融资查询降为 15% 配额 |
| **编辑器审核 CLI** | `python3 crawler/tools/curate.py` 一次一个候选展示，`y/n/s/q` 按键操作 |
| **大厂质量门槛** | Prompt 规则：知名品牌的产品只有在代表"新能力 / 新行为"时才算合格，版本号迭代、定价、本地化不算 |

### P1（最好有）

- **Hook 多语言展示**：`weird_form` → "形态奇特"，`new_behavior` → "新行为" 等
- **Hero 配图自动抓取**：从产品官网 og:image 拿一张大图作为卡片背景
- **每日推送**：编辑器审完后触发邮件 / Slack 通知（自用）

### P2（v3+ 再考虑）

- 👍 / 👎 反馈按钮 + 个人化品味画像
- 历史 hero 时间线（"过去 30 天每日产品"）
- 多语言版本（目前中文为主即可）

## 七、数据模型变更（增量、不破坏旧数据）

**新增字段（向后兼容）**：
- `screenshot_worthy: bool` — 是否够格当 hero
- `hook: enum` — `weird_form` / `new_behavior` / `unexpected_combo` / `quiet_real_problem` / `new_interaction` / `niche_depth`

**弃用字段（保留不删）**：
- `dark_horse_index`、`criteria_met`、`rising_star` 分层 — 老数据仍带，但前端不再用

**MongoDB 同步**：增量字段自动跟进，不需要 migration。

## 八、成功指标

### Leading Indicators（1-4 周内能看到）

- **每日选出 hero 的成功率**：候选池里编辑器能挑出"自己愿意截图"的比例 ≥ 60%
- **Hero 点击率**：访客中点进 hero 详情页的比例 ≥ 30%
- **首页跳出率**：从首页直接关闭的比例 ≤ 50%
- **编辑器耗时**：每天审完候选 ≤ 10 分钟

### Lagging Indicators（1-3 月）

- **小米团队主动分享数**：有人在群里转 hero 链接 ≥ 每周 2 次
- **回访率**：注册用户 / 书签用户每周回访 ≥ 3 次
- **编辑器的"今天有 oh shit 吗"日记**：连续 30 天能写出"今天的 oh shit 是什么、为什么" ≥ 25 / 30 天

## 九、未决问题

- **[编辑器]** 编辑器每天审 5 分钟可持续吗？周末怎么办？需要预选机制还是允许跳过？
- **[抓取]** Makuake / 摩点 / 小米有品的搜索结果能不能稳定通过 Perplexity / GLM 拿到？需要打样验证。
- **[运营]** 首页只有一个 hero 时，加载页面但还没今日 pick 的兜底是什么？昨天的？最近 7 天里点击率最高的？
- **[产品]** Recent picks 是否需要"已读"标记？还是每次刷新都按时间倒序？
- **[设计]** Hero 卡片在没有产品配图时怎么处理？纯文字 + 渐变背景？

## 十、风险

| 风险 | 缓解 |
|---|---|
| 编辑器疲劳（一个人每天选） | CLI 工具体验做到极致快（5 分钟内审完）；周末允许跳过 |
| 抓取池找不到够"奇怪"的产品 | 副池（大厂 release）+ 主池（众筹）双保险；持续调 prompt |
| 二元标签太粗糙，丢失信息 | `hook` enum 保留细分维度；后期可加权重 |
| 旧用户依赖黑马 / 潜力股 API | 旧端点保留但 deprecated，路由到新过滤逻辑 |

---

## 十一、阶段

- **Phase 1（subtraction，1-2 周）**：完成 Claude Code 改造单——重写 prompt、首页改版、curate.py、API 增量端点
- **Phase 2（taste calibration，2-4 周）**：每天用、记日记、根据"我自己嫌弃"的 pick 反向调 prompt
- **Phase 3（外部用户，4-8 周）**：让 5-10 个小米同事 + AI 朋友日常使用，收口头反馈
